Introduction¶

The goal of this project is to compare the effectiveness of tf-idf and Doc2vec on differentiating different roasts of coffee from a short description. My reason for pursuing this project is that I have always looked at the different coffees in grocery stores and coffee shops and noticed that the descriptions seem to give away what kind of roast the coffee is. For example, seeing words like rich or chocolate usually indicate a darker roast while words like fruit indicate a lighter roast. In this project, my goal is to find out whether that is a generalization, or whether you can reliably predict a coffee roast based on the description and flavor profiles of the coffee.

Gather the Data¶

The first thing I had to do was gather data. I am sure that there is some dataset already curated online, but I really wanted to scrape the data myself. I looked at a bunch of different coffee sites and was getting quite discouraged since each of them only had 10-15 different coffees. Eventually, though, I realized that a coffee subscription service probably has the most different and unique coffees. I found www.mistobox.com which has over 600 different coffees available! That is perfect for this project, the only thing that I have to do is get the data in here now.

I originally planned to use BeautifulSoup4 to scrape the website, however, I quickly noticed an issue. When loading the page on a browser (or in bs4) it only loaded 12 different coffees at a time, and in order to load the rest, you had to scroll down. The solution to this problem could have been something like selenium, which you can control like a browser (i.e. make it scroll down and load all the coffees). However, I have never used selenium from within a jupyter notebook and it also is a little heavy-handed for a data scraping task like this. I then checked out the network tab of Google Chrome and noticed that when you scrolled down, it was making an API call. This seemed like a request that I could make using the python requests library. Below is the code that I eventually got to work for scraping the data.

    "accept": "application/json",
    "accept-language": "en-US,en;q=0.9",
    "content-type": "application/x-www-form-urlencoded",
    "sec-ch-ua": "\" Not A;Brand\";v=\"99\", \"Chromium\";v=\"96\", \"Google Chrome\";v=\"96\"",

s = requests.Session()
s.headers.update({'referer': "https://www.mistobox.com/"})

# Store data and labels in arrays with matching indices
data = []
labels = []

# There are 55 "pages" of coffee so we call 55 times
for i in range(1, 55):
    # Make a post request
    resp = s.post(url, data="{\"params\":\"query=&filters=searchable%3Atrue&hitsPerPage=12&page=" + str(i) + "&facets=%5B%22*%22%5D\"}", headers=headers)
    for hit in resp.json()['hits']:
        # Format the label and the description nicely
        desc = hit['short_description'].replace(',', '')
        flavors = ' '.join([x.lower() for x in hit['flavors']])
        total_data = desc + ' ' + flavors
        
        lab = '-'.join([x.lower() for x in hit['roasts']])
        
        # Only add data if both label and description are present 
        if total_data != '' and lab != '':
            data.append(total_data)
            labels.append(lab)

# Function to generate frequency graph of different words for roast of coffee
def generate_graph_for_words(words):
    graph = [0, 0, 0, 0, 0]
    possible_labels = ['light', 'light-medium', 'medium', 'medium-dark', 'dark']

    # Look through descriptions and labels and increment count for label if word is in description
    for x, y in zip(data, labels):
        for word in words:
            if word in x:
                graph[possible_labels.index(y)] += 1
                break

    # label axis and plot graoh
    fig = plt.figure(figsize=(20,10))
    plt.bar(possible_labels,graph)
    plt.ylabel(f'Frequency of Word{"s" if len(words) > 1 else ""} {", ".join([x.capitalize() for x in words])}', fontsize=20)
    plt.xlabel('Roast of Coffee', fontsize=20)
    plt.title(f'Frequency of Word{"s" if len(words) > 1 else ""} {", ".join([x.capitalize() for x in words])} vs Roast of Coffee', fontsize=25)
    plt.show()

The function above essentially counts how many coffee descriptions contain a specific word, partitioned by the roast of the coffee. This is a very important function as it can help to show if my predictions about different words and their trends are true.

tf-idf vs Doc2vec¶

Now that we have done some fun EDA, lets get into the really fun stuff, tfidf vs Doc2vec. After looking at the graphs above, I am pretty concerned about their performance. I really thought that there was going to be specific words that more obviously corresponded to different roasts of coffee, but that did not exactly seem to be the case.

tf-idf¶

We are going to start with tf-idf. This essentially counts how many times a word appears in a document as well as how many documents the word appears in. The general idea is that words that appear in many documents are less important than words that appear in only a few documents. When evaluating the model on the test set, we will be using cosine similarity to find the most similar document from the test set and using that label as our answer.

# Split into initial train/test data
train_x, train_y = data[:400], labels[:400]
test_x, test_y = data[400:], labels[400:]

# Train the Doc2Vec model
tagged_data = [TaggedDocument(words=word_tokenize(_d.lower()), tags=[str(i)]) for i, _d in enumerate(train_x)]
model = Doc2Vec(tagged_data)


# Now check accuracy of test data
correct = 0
for x, y in zip(test_x, test_y):
    test = word_tokenize(x)
    ivec = model.infer_vector(test)
    g = model.dv.most_similar(positive=[ivec], topn=1)
    if train_y[int(g[0][0])] == y:
        correct += 1

print(f'accuracy = {correct/len(test_y)}')

acc_tfidf = []
acc_doc2vec = []

for i in range(10):
    # 2/3 1/3 split
    X_train, X_test, y_train, y_test = train_test_split(data, labels, test_size=0.33)
    
    # train and test tfidf
    vectorizer = TfidfVectorizer()
    X = vectorizer.fit_transform(X_train).toarray()

    # Now check accuracy of test data
    correct = 0
    for x, y in zip(X_test, y_test):
        test_vec = vectorizer.transform([x]).toarray()
        temp = X.dot(test_vec.T)

        for mat in temp.T:
            temp2 = [(y_train[i], x) for i, x in enumerate(mat)]
            if sorted(temp2, key=lambda i: i[1], reverse=True)[0][0] == y:
                correct += 1

    acc_tfidf.append(correct/len(test_y))
    
    # train and test doc2vec
    tagged_data = [TaggedDocument(words=word_tokenize(_d.lower()), tags=[str(i)]) for i, _d in enumerate(X_train)]
    model = Doc2Vec(tagged_data)


    # Now check accuracy of test data
    correct = 0
    for x, y in zip(X_test, y_test):
        test = word_tokenize(x)
        ivec = model.infer_vector(test)
        g = model.dv.most_similar(positive=[ivec], topn=1)
        if y_train[int(g[0][0])] == y:
            correct += 1

    acc_doc2vec.append(correct/len(test_y))
